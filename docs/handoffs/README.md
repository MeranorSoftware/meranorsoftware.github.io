# Thread Handoff Process

Use this folder whenever work is ready to move into a new Codex thread.

## Handoff Checklist

Each handoff note should include:

1. Completed work
2. Locked brand or architecture decisions
3. Commands run and the outcome of each
4. Known issues or limitations
5. Exact recommended next steps
6. A ready-to-paste opening prompt for the next thread

## Naming

Use a timestamped filename such as:

- `2026-07-03-homepage-v0.1-handoff.md`

## When To Create One

- When a milestone is complete and a clean context split would help
- When the thread approaches practical context limits
- When the next phase changes from foundation work to a different area such as product pages, content tooling, or deployment
